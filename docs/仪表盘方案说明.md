# 仪表盘优化方案说明

## 已实施方案：方案二（部分实现）- 不同角色显示不同的仪表盘内容

### 当前实现特点：
1. **所有角色都可以访问仪表盘**：显示公共无敏信息
2. **角色差异化显示**：
   - **系统管理员**：显示全部统计信息（包括系统用户数）
   - **其他角色**：隐藏敏感信息（系统用户数），显示其他公共统计

### 内容划分：

#### 所有角色可见（公共信息）：
- 统计卡片：养殖计划、产量统计、养殖区域、养殖品种、饲料种类、销售记录、待审批计划（有权限者）
- 未读消息（个人消息）
- 最近计划（根据权限过滤）
- 功能模块入口（根据权限显示）
- 系统信息（基础信息，无敏感数据）

#### 仅管理员可见：
- 系统用户数统计卡片
- 最近操作日志

### 布局优化：
✅ **已完成**：
- 系统信息卡片移至最底部，横向排列
- 优化卡片高度，减少最小高度限制
- 使用更紧凑的布局，避免超出视口出现滚动条
- 响应式设计，移动端自动调整布局

## 其他可选方案

### 方案一：只有系统管理员能查看仪表盘（最简单）
- **优点**：实现简单，权限控制清晰，不泄露敏感信息
- **缺点**：其他角色无法看到仪表盘，用户体验较差
- **适用场景**：对数据安全要求极高的场景
- **实现方式**：在路由守卫中判断角色，非管理员跳转到其他页面

### 方案三：仪表盘只显示公共无敏信息，所有角色都能查看（更严格）
- **优点**：实现简单，所有用户都能看到，无权限风险
- **缺点**：功能受限，对管理员不够友好
- **内容**：
  - 未读消息（个人）
  - 最近计划（只显示自己创建的或可查看的）
  - 功能模块入口
  - 系统信息（仅基础信息，不含敏感数据）
  - 不显示任何统计数据（避免泄露信息）

### 方案四：完全角色定制化仪表盘（最灵活）
- **优点**：每个角色看到最适合自己的内容，体验最佳
- **缺点**：开发工作量最大，需要维护多套模板
- **内容划分**：
  - **系统管理员**：全系统统计、用户管理、系统监控
  - **部门管理员**：部门统计、部门计划、部门人员
  - **普通操作员**：我的计划、我的任务、我的消息
  - **决策层**：汇总报表、趋势分析、审批事项
