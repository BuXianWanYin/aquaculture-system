# 水产养殖管理系统 - 技术栈文档

## 一、项目概述

本项目采用前后端分离架构，后端使用 Spring Boot 框架，前端使用 Vue 3 框架。

## 六、核心技术特性

### 6.1 后端特性

1. **RESTful API 设计**
   - 统一的接口规范
   - 标准的 HTTP 方法（GET、POST、PUT、DELETE）

2. **JWT 身份认证**
   - 无状态的用户认证机制
   - Token 自动刷新

3. **AOP 切面编程**
   - 自动记录操作日志
   - 统一的异常处理

4. **MyBatis Plus 增强**
   - 自动生成 CRUD 操作
   - 分页插件支持
   - 条件构造器简化查询

5. **Redis 缓存**
   - 提升系统性能
   - 会话管理

6. **参数验证**
   - 使用 Spring Validation 进行参数校验
   - 统一异常处理

### 6.2 前端特性

1. **组件化开发**
   - 基于 Vue 3 Composition API
   - 可复用的组件设计

2. **路由管理**
   - 基于 Vue Router 4 的路由系统
   - 路由守卫实现权限控制

3. **状态管理**
   - 使用 Pinia 进行全局状态管理
   - 用户信息、权限信息统一管理

4. **UI 组件库**
   - Element Plus 提供丰富的 UI 组件
   - 统一的视觉风格

5. **数据可视化**
   - ECharts 实现图表展示
   - 支持多种图表类型

6. **HTTP 请求封装**
   - Axios 统一封装
   - 请求拦截器（Token 注入）
   - 响应拦截器（统一错误处理）

## 七、开发规范

### 7.1 后端开发规范

1. **包命名规范**
   - Controller: `com.server.aquacultureserver.controller`
   - Service: `com.server.aquacultureserver.service`
   - Mapper: `com.server.aquacultureserver.mapper`
   - Domain: `com.server.aquacultureserver.domain`

2. **类命名规范**
   - Controller: `*Controller`
   - Service: `*Service` / `*ServiceImpl`
   - Mapper: `*Mapper`
   - Domain: 实体类名（如 `SysUser`）

3. **接口设计规范**
   - 统一返回格式: `Result<T>`
   - RESTful 风格 URL
   - 统一异常处理

4. **代码规范**
   - 使用 Lombok 简化代码
   - 统一使用 MyBatis Plus 进行数据库操作
   - 使用 AOP 记录操作日志

### 7.2 前端开发规范

1. **文件命名规范**
   - 组件文件: PascalCase（如 `UserList.vue`）
   - 工具文件: camelCase（如 `request.js`）
   - API 文件: camelCase（如 `user.js`）

2. **组件规范**
   - 使用 Composition API
   - 组件单一职责
   - Props 和 Emits 明确类型

3. **代码规范**
   - 使用 ESLint 进行代码检查
   - 统一的代码格式化
   - 注释规范

## 八、构建与部署

### 8.1 后端构建

```bash
# 使用 Maven 打包
mvn clean package

# 运行 JAR 包
java -jar aquaculture-server-1.0.0.jar
```

### 8.2 前端构建

```bash
# 安装依赖
npm install

# 开发环境运行
npm run dev

# 生产环境构建
npm run build

# 预览构建结果
npm run preview
```

### 8.3 部署配置

**后端配置** (`application.yml`):
- 端口: 8080
- 数据库: MySQL (localhost:3306)
- Redis: localhost:6379

**前端配置** (`vite.config.js`):
- 开发端口: 3000
- API 代理: http://localhost:8080
