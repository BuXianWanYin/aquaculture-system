========================================
  数据库导入说明
  Database Import Instructions
========================================

本文件夹包含数据库初始化 SQL 文件：aquaculture_management.sql

使用 Navicat 导入数据库的步骤：

1. 打开 Navicat
   - 启动 Navicat Premium 或其他版本的 Navicat

2. 连接 MySQL
   - 在 Navicat 中点击"连接" -> "MySQL"
   - 输入连接信息：
     * 主机：localhost（或您的 MySQL 服务器地址）
     * 端口：3306（默认端口）
     * 用户名：root（或您的 MySQL 用户名）
     * 密码：输入您的 MySQL 密码
   - 点击"测试连接"确认连接成功
   - 点击"确定"保存连接

3. 新建数据库
   - 在左侧连接列表中，找到刚创建的 MySQL 连接
   - 右键点击连接，选择"新建数据库"
   - 数据库名：aquaculture_management
   - 字符集：utf8mb4
   - 排序规则：utf8mb4_unicode_ci（或 utf8mb4_general_ci）
   - 点击"确定"创建数据库

4. 运行 SQL 文件
   - 在左侧连接列表中，展开 MySQL 连接
   - 找到并点击新建的数据库 "aquaculture_management"
   - 右键点击数据库 "aquaculture_management"
   - 选择"运行 SQL 文件..."（或"Execute SQL File"）

5. 选择 SQL 文件
   - 在弹出的文件选择对话框中
   - 浏览到本文件夹（sql 文件夹）
   - 选择文件：aquaculture_management.sql
   - 点击"打开"

6. 开始运行
   - 确认 SQL 文件路径正确
   - 点击"开始"按钮执行 SQL 文件
   - 等待执行完成（可能需要几秒到几分钟，取决于文件大小）

7. 验证导入结果
   - 执行完成后，查看执行日志确认没有错误
   - 刷新数据库（右键点击数据库 -> "刷新"）
   - 检查数据库表是否都已创建成功
   - 应该能看到多个表，如：
     * sys_user（用户表）
     * sys_role（角色表）
     * sys_permission（权限表）
     * base_department（部门表）
     * base_area（区域表）
     * 等等...

========================================
注意事项：
========================================

1. 确保 MySQL 版本为 8.0 或更高版本
2. 确保已安装并启动 MySQL 服务
3. 确保有足够的权限创建数据库和表
4. 如果数据库中已存在同名表，导入可能会报错
   - 解决方案：先删除已存在的数据库，或修改 SQL 文件中的表名

5. 字符集建议使用 utf8mb4，以支持 emoji 等特殊字符

6. 导入时间取决于：
   - SQL 文件大小
   - 数据库服务器性能
   - 初始数据的多少

7. 如果导入失败：
   - 查看错误日志，根据错误信息排查问题
   - 常见问题：
     * 权限不足 -> 使用有足够权限的用户
     * 数据库已存在 -> 先删除或使用其他数据库名
     * 表已存在 -> 删除已存在的表或数据库
     * 字符集不支持 -> 确保数据库字符集为 utf8mb4

========================================
快速步骤总结：
========================================

1. 打开 Navicat 并连接 MySQL 8
2. 新建数据库：aquaculture_management
3. 选择数据库，右键 -> 运行 SQL 文件
4. 选择本文件夹中的 aquaculture_management.sql
5. 点击"开始"运行即可
